How it works

Three layers. One review.

Each module handles a different dimension of code quality. Together, they catch what neither could alone.

Static AST Analysis

ast_analyser.py

Instant & Free

Parses Python source into an Abstract Syntax Tree and walks it to find problematic patterns — no API calls, no latency, no cost.

Detects string literals matching patterns for API keys, passwords, tokens, and other credentials embedded directly in source code. Catches API_KEY = "sk-..." and similar patterns before they reach production.

Flags calls to eval(), exec(), and compile() — functions that execute arbitrary strings as code, opening the door to injection attacks.

Catches overly broad except: blocks that swallow all exceptions silently, including KeyboardInterrupt and SystemExit, making bugs nearly impossible to diagnose.

Checks that newly defined functions include docstrings. Undocumented functions make codebases harder to maintain and onboard new developers.

ast_analyser.py
import ast
# Pattern: detect hardcoded secrets
SECRET_PATTERNS = [
"api_key", "password",
"secret", "token",
]
class ASTAnalyser:
def scan(self, source: str):
tree = ast.parse(source)
issues = []
for node in ast.walk(tree):
if self._has_secret(node):
issues.append(
"🔑 Secret found")
if self._uses_eval(node):
issues.append(
"⚠️ eval() used")
return issues

AI Code Review

main.py

GPT-4 Powered

LangChain sends your code diff alongside a carefully crafted system prompt to GPT-4. The AI evaluates logic, security, and style with deep semantic understanding that static analysis alone can't provide.

Logic Errors

Catches incorrect conditions, infinite loops, and off-by-one errors that pass syntax checks but fail at runtime. Things like range(len(x)) vs range(len(x)-1).

Security Vulnerabilities

Identifies input validation risks, SQL injection via f-strings, XSS vectors, and other attack surfaces. Understands context — not just patterns.

Best Practices & Style

Naming conventions, readability improvements, redundant operations, and code that works but could be cleaner. The stuff senior engineers would comment on in PR review.

How AST Analysis Works

Python's ast module parses source code into a structured tree of nodes, enabling deep logical queries without running the file.

01 Parse
import ast
tree = ast.parse(source)

Transforms raw source code text into a structured Abstract Syntax Tree of node objects representing logic constructs.

02 Walk
for node in ast.walk(tree):
  self.inspect(node)

Walks node by node in a depth-first traversal, testing each block against predefined lint rules and code patterns.

03 Report
return {
  "line": node.lineno,
  "msg": "Bare except"
}

Compiles all node findings into structured notes, ready to combine with the AI review and post on GitHub.
